# 沙丘內容控制台工程清單 v2

## P0 必修

### 1. 認證一致性
- `staff-dashboard-v2` 啟動時統一以 `/api/me` 確認登入與權限
- 不再只靠 `localStorage.loginUser` 判斷能否操作
- 若登入失效，直接導回登入頁並顯示明確訊息

### 2. 玩法入口主操作重整
- 卡片主按鈕只保留 `管理內容`
- `結構地圖` 保持卡片層直接可見，方便直接檢查主結構
- `編輯 / 刪除` 收進 `⋯` 更多選單
- 刪除前明確提示底下仍有關卡 / 棋盤資料會被一併清理

### 3. 新增 / 編輯 / 刪除回饋清楚化
- 表單送出中、成功、失敗都要有明確訊息
- 成功後列表對應卡片高亮
- 失敗時直接顯示錯誤原因
- 刪除使用一致的確認視窗，不混用原生 `confirm`

### 4. 關卡不可直接跨劇情共用
- 劇情主線關卡只能屬於一個 `quest_chain`
- 編輯既有關卡時，不允許直接把 `quest_chain_id` 改到另一條劇情
- 若要重用內容，改成 `複製關卡`

## P1 體驗優化

### 5. 關卡複製機制
- 關卡卡片可直接 `複製`
- 複製會建立一顆新的 task，不污染原關卡
- 複製後預設留在目前主結構底下

### 6. 新增關卡改成分步驟 Wizard
- 第一步：選內容類型
- 第二步：填基本資料
- 第三步：填玩法與驗證
- 第四步：填玩家文案與素材

### 7. 刪除影響範圍提示 + 一次性清理
- 刪除玩法入口前提示：
  - 關聯關卡數
  - 大富翁棋盤數
  - 棋盤格子數
  - 玩家劇情進度數
  - 玩家關卡進度數
- 管理員確認後，系統直接 cascade 刪除該入口底下的子內容
- 不要求管理員手動回頭拆每一個關聯來源
- 刪除關卡前提示：
  - 是否仍綁定棋盤格
  - 是否已有玩家進度

### 7.1 舊資料清理基線
- 正式資料庫只保留兩個教學入口：
  - `沙丘教學模式｜手機流程導覽`
  - `沙丘教學模式｜大富翁流程導覽`
- 任何 V2 驗證流程產生的臨時劇情，驗證後必須清除
- 驗證前後都要重新查核 `quest_chains`，避免舊入口污染判斷

## P2 結構升級

### 8. 主結構可視化地圖
- 選一個玩法入口後，可看到一張結構地圖
- 劇情主線顯示關卡順序與節點關係
- 大富翁顯示棋盤格子順序與類型

### 9. 節點詳情
- 點節點後右側顯示：
  - 關卡 / 格子名稱
  - 類型
  - 對應 NPC
  - 道具需求 / 獎勵
  - 音效 / BGM
  - 驗證方式

### 10. NPC / 道具 / 音效 / 驗證關聯
- 劇情關卡透過任務欄位與模板資訊顯示
- 大富翁格子透過 tile 類型與綁定任務顯示
- 先提供第一版關係可視化，再逐步補強成完全可編輯

## 驗收流程

### 除錯經理
- 先檢查流程是否能真的登入
- 檢查列表更新是否即時
- 檢查關卡是否會跨劇情污染

### 技術經理
- 核對 API、權限、欄位與資料來源是否一致
- 確保前後端錯誤訊息可追蹤

### 資深工程驗證
- 真實新增玩法入口
- 真實編輯玩法入口
- 真實新增關卡
- 真實編輯關卡
- 真實刪除關卡 / 玩法入口
- 真實檢查結構地圖
- 驗證完後再次清理測試資料，並回查遠端 DB 僅保留教學入口

### 專案經理驗收
- 人：管理員是第一次使用時是否看得懂主操作
- 事：新增 / 編輯 / 刪除 / 查看結構圖是否順
- 時：操作後是否立刻看得見結果
- 地：在哪個主結構底下是否一眼可辨
- 物：NPC / 道具 / 音效 / 驗證方式是否能被看見
- 驗收補充：
  - 入口卡是否能直接看到 `管理內容 / 結構地圖`
  - 刪除玩法入口時，是否不需要再回頭手動拆子內容
